# コーディング規約

## 基本原則

- **Pythonic**: PEP 8 に準拠したクリーンなコードを書く。
- **型安全性**: `mypy` による厳格な型チェックをパスすること。
- **非同期処理**: `asyncio` を適切に使用し、I/O ブロッキングを避ける。

## 命名規則

- クラス名: `PascalCase`
- 関数・変数名: `snake_case`
- 定数: `UPPER_SNAKE_CASE`

## フォーマット

- `black` を使用して自動フォーマットを行う（1 行の長さは 120 文字）。
- インポート順序は `isort`（または `black` のデフォルト設定）に従う。

## ドキュメント

- 公開関数やクラスには Google スタイルの docstring を記述する。
- 複雑なロジックにはコメントを付与し、意図を明確にする。

## Pydantic の利用

- 外部からの入力および WebSocket で送信するデータには Pydantic モデルを使用し、バリデーションを強制する。
