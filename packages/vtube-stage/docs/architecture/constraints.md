# 制約事項と既知の課題

## 技術的制約

- **ブラウザの自動再生ポリシー**: ユーザーが画面を一度クリックするまで、TTS による音声再生がブロックされる場合があります。
- **GPU 依存**: 3D レンダリングを行うため、実行環境には一定以上の GPU 性能が必要です。
- **VRM バージョン**: 現在は VRM 1.0 規格を主に対象としています。VRM 0.x モデルは互換性レイヤーを通じて動作しますが、一部の挙動が異なる可能性があります。

## 既知の課題

- **メモリ消費**: 複数の VRM モデルを頻繁に切り替えると、メモリ消費量が増大する可能性があります。
- **TTS 遅延**: VOICEVOX への HTTP リクエストが遅い場合、発話の開始が遅れたり、リップシンクと音声の同期がずれることがあります。
- **speak の完了同期**: `speak` ツールは renderer からの `speakEnd` を受けるまで待機します (タイムアウト 30 秒)。renderer 側が `speakEnd` を返さない場合、AI 側がブロックする可能性があります。

## 開発上の注意

- **環境変数の名前**: 内蔵 MCP サーバーの設定 `STAGE_DIRECTOR_MCP_HOST` / `STAGE_DIRECTOR_MCP_PORT` (既定 127.0.0.1:8080) には、削除された旧パッケージ `stage-director` に由来する名前が歴史的経緯で残っています。
