# 機能テスト結果テンプレート

## 概要

受け入れ基準に基づく詳細な機能テスト結果を記録するテンプレートです。

## 完了条件

- [ ] すべてのチェック項目が実行されている。
- [ ] 正常系と異常系が確認されている。
- [ ] 結果が記録されている。
- [ ] 発見された課題が対応されている。
- [ ] 確認結果がチケットに記録されている。

## 構造

```markdown
# [Test] 機能テスト結果: {機能名}

## テスト環境

- バージョン: {コミットハッシュ / バージョン}
- OS/ブラウザ: {環境情報}
- 日付: {YYYY-MM-DD}
- 実行者: {名前}

## 対象

テストケース（Git）: `docs/specs/{機能名}/test-specs/spec.md`

## 結果サマリー

| 対象      |  合計  |   OK   |  NG   | スキップ |   結果   |
| :-------- | :----: | :----: | :---: | :------: | :------: |
| 正常      |   10   |   10   |   0   |    0     |   PASS   |
| 異常      |   5    |   5    |   0   |    0     |   PASS   |
| **Total** | **15** | **15** | **0** |  **0**   | **PASS** |

## 詳細結果（NG/Skip のみ）

| Case ID          | 結果 | ログ/補足             |
| :--------------- | :--- | :-------------------- |

## 備考

- 特記事項
```
