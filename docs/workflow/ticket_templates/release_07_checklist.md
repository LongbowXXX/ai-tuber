# タスク 7: リリース確認項目作成

[← テンプレート一覧に戻る](./README.md)

## チケットタイトル

`[リリース確認項目] v{X.Y.Z}`

## チケット内容

```
## リリースバージョン
v{X.Y.Z}

## 確認項目

### 動作確認
- [ ] {機能ストーリー1}のExitCriteria確認
- [ ] {機能ストーリー2}のExitCriteria確認
- [ ] {修正内容}の動作確認

#### 品質・セキュリティ
- [ ] 脆弱性チェック完了
- [ ] 静的解析完了
- [ ] ライセンス確認完了

#### ドキュメント
- [ ] README/CHANGELOG更新
- [ ] API仕様書更新
- [ ] 依存ライブラリリスト更新

### リリースアーティファクト
- [ ] ビルド成果物作成
- [ ] サンプルコード/データ準備

### バージョン管理
- [ ] ソースコード内のバージョン番号更新
- [ ] package.json/pyproject.toml更新

## 確認
- [ ] すべての確認項目が網羅されている
- [ ] 確認項目が具体的で実行可能である
```

## 使い方

1. `{X.Y.Z}` を実際のバージョン番号に置き換える
2. 動作確認項目に含まれる機能ストーリーをリストアップ
3. 品質・セキュリティチェックの完了を確認項目に追加
4. ドキュメント更新を確認項目に追加
5. リリースアーティファクトの準備を確認項目に追加
6. 確認項目をチケットに記録

- **前提**: タスク 1（リリースバージョン・タグ名合意）
- **後続**: タスク 8（リリース前確認実施）
