# タスク 5: 機能実装完了確認項目作成

[← テンプレート一覧に戻る](./README.md)

## チケットタイトル

`[確認項目] {機能名}`

## チケット内容

```
## 対象機能
{機能名}

## 確認項目
### 正常系
- {正常系テストケース1}
- {正常系テストケース2}

### 異常系
- {異常系テストケース1}
- {異常系テストケース2}

### 境界値
- {境界値テストケース1}

## 確認
- [ ] すべての仕様項目に対応する確認項目がある
- [ ] 正常系・異常系の両方が網羅されている

## 成果物
- [ ] 確認項目(プロジェクト管理ツール上に記録)
```

## 使い方

1. `{機能名}` を実際の機能名に置き換える
2. 仕様をもとに正常系・異常系・境界値のテストケースを作成
3. すべての仕様項目がカバーされていることを確認
4. 確認項目をプロジェクト管理ツール上に記録

## 依存関係

- **前提**: タスク 3（設計）
- **後続**: タスク 8（機能実装完了確認実施）
