# タスク 4: 静的解析

[← テンプレート一覧に戻る](./README.md)

## チケットタイトル

`[静的解析] v{X.Y.Z}`

## チケット内容

```
## リリースバージョン
v{X.Y.Z}

## 使用ツール
{使用する静的解析ツール名（例: Coverity, SonarQube, CodeQL, Pylint, ESLint等）}

## スキャン結果
### Critical
{結果と対応}

### High
{結果と対応}

### Medium/Low
{結果（対応不要の場合は理由を記載）}

## False Positive
{False Positiveと判定した項目と理由}

## 確認
- [ ] 静的解析が実行されている
- [ ] Criticalな問題がすべて対応されている
- [ ] 対応しない問題の理由が記録されている

## 成果物
- [ ] チェック結果(プロジェクト管理ツール上に記録)
```

## 使い方

1. `{X.Y.Z}` を実際のバージョン番号に置き換える
2. 使用する静的解析ツールを記載（Coverity, SonarQube, CodeQL, Pylint, ESLint 等）
3. 静的解析を実行
4. Critical な問題は必ず対応
5. High な問題も原則対応
6. Medium/Low は対応不要の場合理由を記録
7. False Positive は理由を記録
8. チェック結果をプロジェクト管理ツール上に記録
