# タスク 3: 設計

[← テンプレート一覧に戻る](./README.md)

## チケットタイトル

`[設計] {機能名}`

## チケット内容

```
## 対象仕様
{仕様チケット番号}

## 設計内容
### アーキテクチャ
{アーキテクチャの概要}

### インターフェース設計
- API: {APIエンドポイント}
- データモデル: {データ構造}

### 技術スタック
- {使用する技術1}
- {使用する技術2}

## 考慮事項
- [ ] 既存システムとの整合性
- [ ] パフォーマンスへの影響
- [ ] エラーハンドリング
- [ ] テスト容易性

## レビュー
- [ ] チームレビュー完了
- [ ] 設計書をGitにコミット

## 成果物
- [ ] 設計書（`docs/design/{機能名}.md`）

## 注意事項
仕様の不備や不明点があれば憶測で行動せず確認すること。
仕様見直しや改善が必要な場合はチケット化すること。
```

## 使い方

1. `{機能名}` を実際の機能名に置き換える
2. `{仕様チケット番号}` をタスク 1 のチケット番号に置き換える
3. アーキテクチャ、インターフェース、技術スタックを具体的に記載
4. 考慮事項をレビュー
5. 設計書を作成し、Git にコミット

## 依存関係

- **前提**: タスク 1（仕様作成・合意）
- **後続**: タスク 4（サニティテスト項目更新）、タスク 5（機能実装完了確認項目作成）、タスク 6（実装・テスト）
