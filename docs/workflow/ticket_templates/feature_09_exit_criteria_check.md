# タスク 9: ExitCriteria 確認

[← テンプレート一覧に戻る](./README.md)

## チケットタイトル

`[ExitCriteria確認] {機能名}`

## チケット内容

```
## 対象ストーリー
{機能追加・変更ストーリーのチケット番号}

## ExitCriteria確認結果
- [ ] ExitCriteriaのすべての項目が確認されている
- [ ] すべての成果物がGit上に存在する
- [ ] ドキュメントが最新の状態に更新されている
- [ ] チームの承認が得られている

## 確認コメント
{各ExitCriteria項目の確認結果詳細}

## 完了判定
- [ ] ストーリー完了
```

## 使い方

1. `{機能名}` を実際の機能名に置き換える
2. `{チケット番号}` を親ストーリーのチケット番号に置き換える
3. タスク 2 で作成した ExitCriteria 文書を参照
4. すべての項目が満たされているか確認
5. 確認結果をチケットのコメントに記録
6. すべてクリアしていればストーリー完了

## 依存関係

- **前提**: タスク 2（ExitCriteria 作成）、タスク 7（サニティテスト実施）、タスク 8（機能実装完了確認実施）
- **後続**: なし（ストーリー完了）
