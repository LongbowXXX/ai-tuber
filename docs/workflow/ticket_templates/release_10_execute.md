# タスク 10: リリース実施

[← テンプレート一覧に戻る](./README.md)

## チケットタイトル

`[リリース実施] v{X.Y.Z}`

## チケット内容

````
## リリースバージョン
v{X.Y.Z}

## 実施内容
### Gitタグ作成
```bash
git tag -a v{X.Y.Z} -m "Release version {X.Y.Z}"
git push origin v{X.Y.Z}
```

### GitHubリリースページ
- [ ] リリースページ作成
- [ ] リリースノート記載
- [ ] アーティファクトアップロード

### パッケージレジストリ公開
- [ ] PyPI公開（該当する場合）
- [ ] npm公開（該当する場合）

### リリース案内
- [ ] チーム通知
- [ ] ユーザー通知

## 確認
- [ ] Gitタグが作成され、プッシュされている
- [ ] GitHubリリースページが作成されている
- [ ] リリースノートが記載されている
- [ ] アーティファクトがアップロードされている
- [ ] パッケージレジストリに公開されている
- [ ] リリース案内が送信されている
```
````

## 使い方

1. `{X.Y.Z}` を実際のバージョン番号に置き換える
2. Git タグを作成してプッシュ
3. GitHub リリースページを作成
4. リリースノートを記載（タスク 1 で作成した骨子を活用）
5. アーティファクトをアップロード
6. 該当する場合はパッケージレジストリに公開
7. チームとユーザーに通知

## 依存関係

- **前提**: タスク 8（リリース前確認実施）、タスク 9（ExitCriteria 作成）
- **後続**: タスク 11（ExitCriteria 確認）
