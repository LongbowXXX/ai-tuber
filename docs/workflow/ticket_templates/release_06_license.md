## 使い方

1. `{X.Y.Z}` を実際のバージョン番号に置き換える
2. 使用するツールを記載
3. 依存ライブラリのライセンス一覧を取得
   - Python: `pip-licenses --format=markdown > docs/licenses.md`
   - Node.js: `npx license-checker --markdown > docs/licenses.md`
4. ライセンス一覧ファイルを Git にコミット
5. ライセンスを分類（商用可能/要確認/不明）
6. 問題のあるライセンスは対応方法を検討
7. 必要なライセンス表記を更新
8. 更新したファイルを Git にコミット
9. 確認結果をプロジェクト管理ツール上に記録

## 注意事項

### 問題のあるライセンス例

- **GPL 系（GPL-2.0, GPL-3.0, AGPL-3.0）**: コピーレフト条項により、プロジェクト全体のソースコード公開が必要になる可能性
- **ライセンス不明・UNKNOWN**: 法的リスクが不明確

### 一般的に安全なライセンス

- MIT, Apache-2.0, BSD-2-Clause, BSD-3-Clause, ISC

### 確認が必要なライセンス

- LGPL 系: 動的リンクなら通常問題ないが要確認
- MPL-2.0: ファイル単位でのコピーレフト
- CC 系: ソフトウェアには非推奨
