# タスクチケットテンプレート

[← 作業フローに戻る](../workflow.md)

本ディレクトリでは、各タスクのチケット起票時に使用するテンプレートを提供します。プロジェクト管理ツール（JIRA、Redmine、GitHub Issues 等）の種類に応じて、適宜フィールド名や形式を調整してください。

## チケット構造

本プロジェクトでは、以下の階層構造でチケットを管理します:

```
ストーリー（親チケット / Epic）
├─ タスク1（サブチケット / サブタスク / 子チケット）
├─ タスク2（サブチケット / サブタスク / 子チケット）
├─ タスク3（サブチケット / サブタスク / 子チケット）
└─ ...
```

**チケット作成の流れ**:

1. **ストーリーチケットを作成**: 機能追加・変更またはリリースのストーリー全体を表す親チケット
2. **サブチケットを作成**: 各タスクをストーリーチケットのサブチケット（子チケット）として追加
3. **依存関係を設定**: サブチケット間の依存関係を設定（JIRA: 「ブロック/ブロックされている」、Redmine: 「先行/後続」、GitHub Issues: マイルストーンやラベル活用）

## テンプレート使用ガイド

### 基本的な使い方

1. **該当するテンプレートを選択**: タスクに応じたテンプレートを選ぶ
2. **{}で囲まれた部分を置換**: プロジェクト固有の情報に置き換える
3. **不要な項目を削除**: 該当しない項目があれば削除する
4. **追加情報を記載**: 必要に応じて追加情報を記載する

### チケット起票のタイミングと手順

#### 機能追加・変更の場合

1. **親チケット（ストーリー）を作成**: 機能全体を表すストーリーチケット
2. **サブチケットを作成**: 9 つのタスクをサブチケット（子チケット）として追加
3. **依存関係を設定**: サブチケット間の依存関係を設定

#### リリースの場合

1. **親チケット（ストーリー）を作成**: リリース全体を表すストーリーチケット
2. **サブチケットを作成**: 11 のタスクをサブチケット（子チケット）として追加
3. **依存関係を設定**: サブチケット間の依存関係を設定

### チケットの連携

- **親子関係**: すべてのタスクチケットは親ストーリーのサブチケット（子チケット）として作成
- **依存関係**: タスク間の依存関係を設定（各ツールの機能に応じて）
- **関連チケット**: 他のストーリーやチケットへの参照を明記
- **進捗更新**: 作業開始・完了時にステータスを更新し、親チケットにも反映
