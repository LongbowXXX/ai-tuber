# リリースストーリー（親チケット）

[← テンプレート一覧に戻る](./README.md)

## チケットタイトル

`[リリース] v{X.Y.Z}`

## チケット内容

```
## リリースバージョン
v{X.Y.Z}

## バージョニング根拠
- [ ] MAJOR: 破壊的変更
- [ ] MINOR: 後方互換性のある機能追加
- [ ] PATCH: 後方互換性のあるバグ修正

## 含まれる変更
- {機能追加・変更チケット1}
- {機能追加・変更チケット2}
- {バグ修正チケット1}

## リリースノート骨子
### Added
- {新機能}

### Changed
- {変更内容}

### Fixed
- {修正内容}

## 関連タスク
このストーリーは以下のサブチケットで構成されます:
- [ ] リリースバージョン・タグ名合意
- [ ] 関連修正の反映確認
- [ ] 脆弱性チェック
- [ ] 静的解析
- [ ] ドキュメント更新
- [ ] ライセンス確認
- [ ] リリース確認項目作成
- [ ] リリース前確認実施
- [ ] ExitCriteria作成
- [ ] リリース実施
- [ ] ExitCriteria確認

## 完了条件
- [ ] すべてのサブチケットが完了している
- [ ] ExitCriteriaが確認されている
- [ ] リリースが完了している
```

## 使い方

1. `{X.Y.Z}` を実際のバージョン番号に置き換える
2. バージョニング根拠を選択（セマンティックバージョニング）
3. 含まれる変更をリストアップ
4. リリースノート骨子を作成
5. このチケットを親チケットとして作成後、各タスクをサブチケットとして追加
