# タスク 2: 関連修正の反映確認

[← テンプレート一覧に戻る](./README.md)

## チケットタイトル

`[反映確認] v{X.Y.Z}`

## チケット内容

```
## リリースバージョン
v{X.Y.Z}

## 確認対象
### 機能追加・変更
- [ ] {チケット番号}: {機能名}
- [ ] {チケット番号}: {機能名}

### バグ修正
- [ ] {チケット番号}: {修正内容}

### マージ確認
- [ ] すべてのプルリクエストがマージされている
- [ ] ブランチのコンフリクトが解消されている
- [ ] すべての関連チケットがクローズされている

## 確認結果
{詳細な確認結果}
```

## 使い方

1. `{X.Y.Z}` を実際のバージョン番号に置き換える
2. リリースに含まれる機能追加・バグ修正をリストアップ
3. すべてのプルリクエストがマージされているか確認
4. コンフリクトが解消されているか確認
5. 関連チケットがすべてクローズされているか確認
6. 確認結果をチケットに記録

## 依存関係

- **前提**: 各機能ストーリー完了
- **後続**: タスク 3（脆弱性チェック）、タスク 4（静的解析）、タスク 5（ドキュメント更新）
