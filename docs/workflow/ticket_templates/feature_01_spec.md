# タスク 1: 仕様作成・合意

[← テンプレート一覧に戻る](./README.md)

## チケットタイトル

`[仕様] {機能名}`

## チケット内容

```
## 目的
{この機能の目的と背景}

## スコープ
- {含まれる範囲}
- {含まれない範囲}

## ユーザーストーリー
- {ユーザーストーリー1}
- {ユーザーストーリー2}

## 非機能要件
- パフォーマンス: {要件}
- セキュリティ: {要件}
- その他: {要件}

## 合意事項
- [ ] プロダクトオーナーの承認
- [ ] チームレビュー完了
- [ ] 仕様書をGitにコミット

## 成果物
- [ ] 仕様書（`docs/specs/{機能名}.md`）
```

## 使い方

1. `{機能名}` を実際の機能名に置き換える
2. 目的と背景を明確に記述
3. スコープを具体的に定義（含まれるもの/含まれないもの）
4. ユーザーストーリーを記載
5. 非機能要件を検討して記載
6. 成果物として仕様書を作成し、Git にコミット

## 依存関係

- **前提**: なし
- **後続**: タスク 2（ExitCriteria 作成）、タスク 3（設計）
