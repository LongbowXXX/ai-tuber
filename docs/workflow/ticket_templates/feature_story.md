# 機能追加・変更ストーリー（親チケット）

[← テンプレート一覧に戻る](./README.md)

## チケットタイトル

`[機能] {機能名}`

## チケット内容

```
## 概要
{機能の概要説明}

## 目的
{この機能の目的と背景}

## スコープ
- {含まれる範囲}
- {含まれない範囲}

## 関連タスク
このストーリーは以下のサブチケットで構成されます:
- [ ] 仕様作成・合意
- [ ] ExitCriteria作成
- [ ] 設計
- [ ] サニティテスト項目更新
- [ ] 機能実装完了確認項目作成
- [ ] 実装・テスト
- [ ] サニティテスト実施
- [ ] 機能実装完了確認実施
- [ ] ExitCriteria確認

## 完了条件
- [ ] すべてのサブチケットが完了している
- [ ] ExitCriteriaが確認されている
```

## 使い方

1. `{機能名}` を実際の機能名に置き換える
2. `{機能の概要説明}` に機能の簡潔な説明を記入
3. `{この機能の目的と背景}` にビジネス価値や背景を記入
4. スコープを明確に定義する
5. このチケットを親チケットとして作成後、各タスクをサブチケットとして追加
