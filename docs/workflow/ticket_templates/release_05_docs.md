# タスク 5: ドキュメント更新

[← テンプレート一覧に戻る](./README.md)

## チケットタイトル

`[ドキュメント更新] v{X.Y.Z}`

## チケット内容

```
## リリースバージョン
v{X.Y.Z}

## 更新対象
### README.md
- [ ] バージョン番号
- [ ] 新機能の説明
- [ ] 破壊的変更の警告

### CHANGELOG.md
- [ ] Added（新機能）
- [ ] Changed（変更）
- [ ] Deprecated（廃止予定）
- [ ] Removed（削除）
- [ ] Fixed（バグ修正）
- [ ] Security（セキュリティ）

### その他
- [ ] HowToUse/チュートリアル
- [ ] API仕様書
- [ ] 依存ライブラリリスト

## 確認
- [ ] すべてのドキュメントが更新されている
- [ ] バージョン番号が一致している
- [ ] 破壊的変更が明確に記載されている

## 成果物
- [ ] README.md
- [ ] CHANGELOG.md
- [ ] その他のドキュメント
```

## 使い方

1. `{X.Y.Z}` を実際のバージョン番号に置き換える
2. README.md のバージョン番号を更新
3. CHANGELOG.md に変更内容を記載（Keep a Changelog 形式推奨）
4. 破壊的変更がある場合は明確に警告
5. API 仕様書等も必要に応じて更新
6. すべてのドキュメントを Git にコミット
