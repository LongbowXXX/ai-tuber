# タスク 1: リリースバージョン・タグ名合意

[← テンプレート一覧に戻る](./README.md)

## チケットタイトル

`[リリース計画] v{X.Y.Z}`

## チケット内容

```
## リリースバージョン
v{X.Y.Z}

## バージョニング根拠
- [ ] MAJOR: 破壊的変更
- [ ] MINOR: 後方互換性のある機能追加
- [ ] PATCH: 後方互換性のあるバグ修正

## 含まれる変更
- {機能追加・変更チケット1}
- {機能追加・変更チケット2}
- {バグ修正チケット1}

## リリースノート骨子
### Added
- {新機能}

### Changed
- {変更内容}

### Fixed
- {修正内容}

## 合意確認
- [ ] バージョン番号がセマンティックバージョニングに従っている
- [ ] タグ名が命名規則に従っている（v{X.Y.Z}）
- [ ] チームの合意が得られている
```

## 使い方

1. `{X.Y.Z}` を実際のバージョン番号に置き換える
2. セマンティックバージョニングに基づいてバージョン番号を決定
3. 含まれる変更をリストアップ
4. リリースノート骨子を作成
5. チームで合意を得る
6. 合意内容をチケットに記録

## 依存関係

- **前提**: なし
- **後続**: タスク 7（リリース確認項目作成）
