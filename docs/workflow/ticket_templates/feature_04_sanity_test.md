# タスク 4: サニティテスト項目更新

[← テンプレート一覧に戻る](./README.md)

## チケットタイトル

`[テスト項目更新] {機能名}`

## チケット内容

```
## 対象機能
{機能名}

## 更新内容
### 追加項目
- {新規追加する確認項目1}
- {新規追加する確認項目2}

### 更新項目
- {更新する既存項目}

### 削除項目
- {削除する項目（理由も記載）}

## 確認
- [ ] 新機能の動作確認項目が追加されている
- [ ] 既存機能への影響を確認する項目が含まれている
- [ ] 不要な項目が削除/更新されている
- [ ] アプリの最新仕様と一致している
- [ ] チェックリストファイルをGitで管理（大きなファイルはGit LFS使用）

## 成果物
- [ ] 更新されたサニティテスト項目（`docs/test/sanity_checklist.*`）
  - 形式: Excel (`.xlsx`), Markdown (`.md`), CSV (`.csv`) 等、プロジェクトに適した形式
```

## 使い方

1. `{機能名}` を実際の機能名に置き換える
2. 追加・更新・削除する項目を具体的に記載
3. プロジェクトに適したファイル形式でチェックリストを更新
4. 大きなファイル（Excel 等）の場合は Git LFS を使用

- **前提**: タスク 1（仕様作成・合意）、タスク 3（設計）
- **後続**: タスク 7（サニティテスト実施）
