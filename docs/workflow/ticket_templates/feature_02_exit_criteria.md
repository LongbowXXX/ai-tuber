# タスク 2: ExitCriteria 作成

[← テンプレート一覧に戻る](./README.md)

## チケットタイトル

`[ExitCriteria] {機能名}`

## チケット内容

```
## 対象ストーリー
{対象となる機能追加・変更ストーリーのチケット番号}

## ExitCriteria項目
- [ ] 仕様変更が以降の確認項目に反映されている
- [ ] すべての仕様が確定され、合意されている
- [ ] 仕様変更がGit上の資料に反映されている
- [ ] 設計が行われており、設計変更がGit上に反映されている
- [ ] サニティテスト項目が更新されている
- [ ] 機能実装完了確認用の動作確認項目が作成されている
- [ ] 変更されたサニティテスト項目が実施されている

## 成果物
- [ ] ExitCriteria文書(プロジェクト管理ツール上に記録)
```

## 使い方

1. `{機能名}` を実際の機能名に置き換える
2. `{チケット番号}` を親ストーリーのチケット番号に置き換える
3. 必要に応じて ExitCriteria 項目を追加・削除
4. ExitCriteria 文書をプロジェクト管理ツール上に記録
