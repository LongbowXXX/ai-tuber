# タスク 3: 脆弱性チェック

[← テンプレート一覧に戻る](./README.md)

## チケットタイトル

`[脆弱性チェック] v{X.Y.Z}`

## チケット内容

```
## リリースバージョン
v{X.Y.Z}

## チェック対象
- [ ] Python依存ライブラリ（pip-audit/safety）
- [ ] Node.js依存ライブラリ（npm audit）

## チェック結果
### Critical/High
{結果と対応}

### Medium
{結果とリスク評価}

### Low
{結果（記録のみ）}

## 確認
- [ ] すべての依存ライブラリがスキャンされている
- [ ] Critical/Highの脆弱性がすべて対応されている
- [ ] 残存する脆弱性のリスク評価が記録されている

## 成果物
- [ ] チェック結果(プロジェクト管理ツール上に記録)
```

## 使い方

1. `{X.Y.Z}` を実際のバージョン番号に置き換える
2. Python 依存ライブラリを pip-audit または safety でスキャン
3. Node.js 依存ライブラリを npm audit でスキャン
4. Critical/High の脆弱性は必ず対応
5. Medium 以下はリスク評価を記録
6. チェック結果をプロジェクト管理ツール上に記録
