# タスク 6: 実装・テスト

[← テンプレート一覧に戻る](./README.md)

## チケットタイトル

`[実装] {機能名}`

## チケット内容

```
## 対象設計
{設計チケット番号}

## 実装内容
{実装の概要}

## 実装の原則
- [ ] 実装量が多い場合、適切な単位でチケット分割
- [ ] チケット分割時はスケルトンクラスを先に投入
- [ ] 破壊的変更の場合はfeatureブランチを使用

## チェックリスト
- [ ] 設計通りに実装されている
- [ ] コーディング規約に従っている
- [ ] ユニットテストが作成されている
- [ ] テストカバレッジが基準を満たしている
- [ ] コードレビューが完了している
- [ ] CIが通過している

## 成果物
- [ ] ソースコード（`packages/*/src/`）
- [ ] テストコード（`packages/*/tests/`）
```

## 使い方

1. `{機能名}` を実際の機能名に置き換える
2. `{設計チケット番号}` をタスク 3 のチケット番号に置き換える
3. 実装の概要を記載
4. 大きな実装の場合は適切に分割
5. ソースコードとテストコードを作成
6. CI を通過させてマージ

## 依存関係

- **前提**: タスク 3（設計）
- **後続**: タスク 7（サニティテスト実施）、タスク 8（機能実装完了確認実施）
